02 — Problem & Solution

The Challenge

⚡ Peak Demand Spike

Demand for koshais spikes 10–50x normal in a 24-hour window during Eid ul-Adha

📞 No Central Platform

Word-of-mouth only — families rely on neighbors or mosque connections

❌ No-shows Common

Double bookings and no-shows leave families stranded on Eid morning

💰 No Price Transparency

Users often overpay or get quoted unfair rates at the last minute

🔒 No Trust Mechanism

No way to verify a koshai's skill, hygiene, or legitimacy

📅 No Scheduling Tools

Koshais cannot manage multiple bookings efficiently

The Solution

🤝 Two-sided Marketplace

Users book, koshais accept — all inside one app

📅 Slot-based Scheduling

Prevents double booking and manages Eid-morning rush

✅ Verified Badge System

NID + trade certificate review builds trust

💳 In-app Payment

bKash/Nagad eliminates cash ambiguity and reduces no-shows

⭐ Ratings & Reviews

Creates accountability and surfaces top performers

📍 Real-time Tracking

Live availability and booking status give users confidence

05 — Data Architecture

Firestore Schema

Collection: users
users/{uid}
{
  uid: String,           // Firebase Auth UID
  name: String,
  phone: String,          // +8801XXXXXXXXX
  photoUrl: String,
  district: String,
  upazila: String,
  fcmToken: String,
  createdAt: Timestamp,
  referralCode: String,
  credits: Number
}
Collection: koshais
koshais/{uid}
{
  uid: String,
  name: String, phone: String, photoUrl: String,
  nidUrl: String, tradeCertUrl: String,
  district: String, upazila: String,
  ratePerCow: Number, ratePerGoat: Number,
  status: String,        // available | busy | offline
  isVerified: Boolean,
  rating: Number,        // 0.0–5.0
  totalJobs: Number,
  lat: Number, lng: Number,
  surgeMultiplier: Number
}
Collection: bookings
bookings/{bookingId}
{
  userId: String, koshaiId: String,
  address: String, lat: Number, lng: Number,
  date: Timestamp, slot: String,
  animalTypes: { cow: Number, goat: Number },
  rateBreakdown: { total: Number, surgeMultiplier: Number },
  depositPaid: Number,
  status: String  // pending|confirmed|en_route|arrived|completed|cancelled
}
